Why mild overfitting?
| 参数            | 位置         | 当前设置                            | 说明                         |
| ------------- | ---------- | ------------------------------- | -------------------------- |
| `train_dir`   | 全局         | `"./train"`                     | 数据集路径                      |
| `transform`   | 第 38 行     | `Resize(224,224)` + `Normalize` | 仅标准化，无数据增强                 |
| `batch_size`  | 第 25 行     | `16`                            | 每个 batch 的样本数量             |
| `val_size`    | 第 64 行     | `20%`                           | 训练集/验证集比例为 80:20           |
| `shuffle`     | DataLoader | `True`                          | 打乱训练集顺序                    |
| `num_workers` | DataLoader | `0`                             | 数据加载线程数（Colab/Windows 用 0） |
| `pin_memory`  | DataLoader | `True`                          | 优化 GPU 数据加载性能              |

| 参数        | 位置              | 当前设置                               | 说明                |
| --------- | --------------- | ---------------------------------- | ----------------- |
| `model`   | 第 72 行          | `ResNet18`                         | 使用 ImageNet 预训练权重 |
| `weights` | `IMAGENET1K_V1` | 加载预训练参数                            |                   |
| `fc`      | 修改              | `Linear(in_features, num_classes)` | 调整输出类别数（24）       |


| 原因                     | 说明                                        |
| ---------------------- | ----------------------------------------- |
| 1️⃣ 数据集较小              | 仅约 4,500 张训练图像、24 类 → 模型太强                |
| 2️⃣ ResNet18 是全模型微调    | 所有参数都被更新，拟合能力高                            |
| 3️⃣ 缺少正则化/augmentation | 目前 transform 只有 resize + normalize，没有随机扰动 |
| 4️⃣ 无 early stopping   | 继续训练 20 epoch 导致后期记忆训练集噪声                 |
| 5️⃣ learning rate 固定   | 学习率不衰减，后期可能震荡过度更新                         |

Adam 学习率较高，训练速度快但容易过拟合；无正则化使模型更倾向于“记忆训练集”。
